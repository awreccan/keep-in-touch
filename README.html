How it works

A renderer, not a data owner

The app holds no shared bucket of your data — your data lives in your storage, and the app just renders and writes it back. Tap a tier to see how it flows.

🔒 Private by design. The hosted URL is code only — it carries no data. Your contacts never leave your Google account or device. Updating the app never loses your data (it's keyed by project, not by the HTML). Isolate lists with ?project=family, ?project=work. Installable as a PWA on Android & macOS.

Built around it

Behind the scenes — the tooling we built to keep it healthy

The app is one HTML file, but it ships with a small workshop of automation around it: a self-running health monitor, a one-command test harness, an adversarial bug-hunt that converged to zero, and docs that flag themselves when stale. Run the monitor below, then crack open each tool.

🩺 Live-deploy health check
launchd · com.keep-in-touch.healthcheck every 4h
  • 1 VERSION matches HEADlive VERSION equals the repo's HEAD — catches a partial deploy (drift)
  • 2 index.html → HTTP 200the live app on GitHub Pages is actually reachable
  • 3 SW cache = kit-shell-<VERSION>service-worker cache name in sw.js — catches a stale offline cache
✅ All green — stays silent. A macOS notification fires only on drift or outage, so there's no alert fatigue.

Runs every 4 hours and at load via a launchd agent — it survives reboot and session exit, unlike a Claude Code session cron that expires after 7 days and only fires while the REPL is idle. Every run appends a status line to ~/.claude/kit-healthcheck.log.
